SeenShow Downloader — Complete Project Plan
============================================

1. Overview
-----------
A cross-platform GUI application (primary target: Windows .exe) that downloads
videos from seenshow.com. It handles Keycloak OAuth authentication, content
retrieval via the provider's official APIs, and media muxing — all behind a
simple paste-URL-and-click interface.

LEGAL NOTE: Any DRM-related functionality (Widevine license acquisition,
encrypted stream decryption) MUST only be implemented in compliance with
applicable copyright and DRM laws (e.g., DMCA, EU Copyright Directive).
Consult legal counsel before implementing or modifying any DRM workflow.
All DRM operations require explicit licensing from the content provider or
use of the provider's own approved APIs/SDKs.

Platform notes:
  - Windows: distributed as a single PyInstaller .exe; self-update via batch script
  - Linux:   run from source (python run.py); config in ~/.config/SeenShowDL/
  - macOS:   run from source; code-signing & notarization not yet implemented

4. Module Design
----------------

4.1 core/auth.py — SeenAuth
  - Full Keycloak OAuth PKCE flow via NextAuth
  - Flow: CSRF token -> POST signin -> GET Keycloak login page -> POST
    credentials -> follow callback -> extract accessToken from session
  - Accepts on_status callback for UI status updates
  - Raises AuthenticationError with user-friendly messages
  - All requests have 30s timeouts

4.2 core/api.py — SeenAPI
  - get_media_detail(media_id) -> full media data with seasons/episodes
  - get_episode_drm_token(episode_id) -> upfrontToken, MPD URL, key IDs
  - find_episode(media_detail, episode_id) -> single episode dict
  - get_all_episodes(media_id) -> (title, [episodes]) flat list across all
    seasons, sorted by order
  - All API calls go through a centralized request layer that enforces rate
    limits, parses rate-limit response headers (Retry-After, X-RateLimit-*),
    and implements exponential backoff + retry for 429/5xx responses.
    Configurable params: max_requests_per_second (default 5),
    backoff_base (2s), max_retries (3). No raw network call bypasses
    the rate-limiting wrapper.

4.3 core/drm.py — DRM Key Retrieval
  LEGAL NOTE: Do NOT ship, extract, or embed Widevine device (.wvd) files
  in this application. The "pywidevine" library must NOT be used for
  device-file operations. Any DRM key retrieval MUST be done through a
  licensed Widevine CDM or an approved provider API. Consult legal counsel
  before any changes to this module.

  - Fetches MPD manifest, extracts content protection info
  - Obtains decryption keys via the provider's licensed API/CDM
  - DRMtoday headers: x-dt-auth-token (upfrontToken JWT),
    x-dt-custom-data (base64-encoded session metadata)
  - Returns [(kid, key)] tuples

4.4 core/downloader.py — Download Pipeline
  - download_encrypted() — uses yt-dlp Python API with progress_hooks for
    real-time percentage
  - decrypt_file() — calls mp4decrypt via subprocess (resolved by binary_locator)
  - mux_output() — calls ffmpeg via subprocess
  - download_episode() — orchestrates the full pipeline:
    metadata -> keys -> download -> decrypt -> mux -> cleanup
  - Accepts progress_cb(message, percent) and cancel_check() callables
  - Returns DownloadResult(success, output_path, error)
  - All subprocesses use CREATE_NO_WINDOW flag on Windows

4.5 core/url_parser.py
  - Episode URL: seenshow.com/media/{id}/episode/{id}/play
    -> ParsedURL("episode", mid, eid)
  - Series URL: seenshow.com/media/{id}
    -> ParsedURL("series", mid, None)
  - Raises ValueError with example URLs on invalid input

4.6 services/config.py
  - Stores config in %APPDATA%/SeenShowDL/config.json
    (Linux: ~/.config/SeenShowDL/; macOS: ~/.config/SeenShowDL/)
    output_dir, language, remember_credentials
  - Credential helpers: save_credentials(), get_credentials(), clear_credentials()
  - Password stored in OS keyring (Windows Credential Manager / Secret Service)
    via the keyring library — not in the JSON config file
  - Auto-creates directory on first use
  - Unknown config keys are logged but preserved (forward-compatible)

  Privacy & Compliance:
  - GDPR / data subject rights: users may request export or deletion of all
    stored personal data (email, credentials, config) via clear_credentials()
    and a future "Delete Account Data" UI action.
  - Consent: storing email/credentials requires the user to explicitly enable
    remember_credentials; the checkbox text must clearly state what is stored.
  - Data retention: credentials are kept until the user signs out or disables
    remember_credentials. No server-side data retention by this application.
  - Privacy policy / ToS: the app MUST link to a published privacy policy
    and terms of service from the settings screen before production release.
  - Encryption at rest: passwords are stored in the OS keyring (encrypted by
    the OS). config.json itself does NOT contain passwords. If refresh tokens
    or other secrets are added in the future they MUST also use the keyring.
  - Secure deletion: clear_credentials() removes the keyring entry and blanks
    the username from config.json, then calls save(). An audit log entry
    (to LOG_DIR) SHOULD be written on credential deletion for compliance.
  - Testing: unit tests must verify that clear_credentials() leaves no
    residual credential data in config.json or the keyring.

4.7 services/binary_locator.py
  - When frozen (PyInstaller): looks in sys._MEIPASS/binaries/
  - When from source: looks in project_root/binaries/
  - Falls back to system PATH
  - Mandatory SHA256 integrity verification: bundled checksums (embedded in
    the application or loaded from a signed manifest) are compared against
    computed SHA256 for ffmpeg and mp4decrypt at startup. If verification
    fails the application MUST log a clear error naming the failing binary
    and abort (sys.exit(1)) — never fall back to unverified binaries.
    The env vars SEENSHOW_FFMPEG_SHA256 / SEENSHOW_MP4DECRYPT_SHA256 are
    still read as overrides for development, but production builds must
    embed checksums. Results are cached per session.

4.8 services/download_manager.py
  - Accepts list of EpisodeTask objects
  - Runs downloads sequentially in a background threading.Thread
  - Communicates via queue.Queue[StatusUpdate]
  - StatusUpdate: episode_id, phase, progress (0-1), message, error
  - Phases: queued -> downloading -> decrypting -> muxing -> done/error/cancelled
  - Cancel support via threading.Event checked between phases
  - Emits batch_done when all tasks complete
  - Retry with exponential backoff for transient failures (network/timeout/5xx):
      maxRetries = 3 (default), maxBackoffSeconds = 60 (cap per delay),
      backoff formula: min(BACKOFF_BASE ** (attempt+1), maxBackoffSeconds).
      An overall totalTimeoutSeconds (default 600) or circuit-breaker prevents
      indefinite blocking of the download queue. After maxRetries exhausted
      or totalTimeoutSeconds exceeded, the task is marked as failed.
  - Checks available disk space before each task:
      Minimum threshold: max(3 * estimated_task_output_size, 500 MB) free.
      estimated_task_output_size = sum of expected video + audio sizes plus
      temp overhead (~2x for encrypted + decrypted copies).
      Configured via MIN_FREE_DISK_MB (default 500) as a floor.
      If the check fails the task fails fast with a "disk_space_error" and
      the next queued task is attempted (it may target a different volume).
      Units: MB, rounded down. Example: a 1 GB episode needs ~3 GB free.
  - Cleans up partial files on final failure
  - Background thread wrapped in try/except; crashes reported via StatusUpdate

4.9 services/updater.py
  - check_for_update() -> queries api.github.com/repos/{repo}/releases/latest,
    compares version tags; fetches optional .sha256 checksum file from release
  - download_update(url, expected_sha256) -> streams exe to %TEMP% with progress
    callback; computes SHA256 of downloaded file and verifies against expected
    hash if provided; aborts on mismatch
  - apply_update(new_exe) -> verifies new_exe exists, writes a batch script that:
      1. Polls until the current exe is unlocked (up to 30s)
      2. Backs up current exe to .old
      3. Moves new exe into place; on failure, rolls back from .old
      4. Launches new exe with a --verify flag for a health-check
      5. If the health-check exits 0, marks .old as verified-to-delete
      6. If the health-check fails, rolls back (.old -> current) automatically
      7. Deletes the batch script itself only on success
    All paths interpolated into the batch script are escaped via
    batch_escape() to handle %, ^, &, |, <, >, and ! characters.
  - cleanup_old_exe() -> called on startup, only removes .old files that
    are older than a safe grace period (e.g., 1 hour) or are explicitly
    marked as verified-to-delete (via a .old.verified sentinel file).
    If .old exists without .old.verified, the updater logs a warning and
    leaves the file for manual inspection or automatic rollback.

6. Threading Model
------------------
Main Thread (tkinter)              Background Thread
      |                                  |
      +-- on_fetch() ------------------>| fetch episode list from API
      |<-- after(0, _populate) ---------|
      |                                  |
      +-- on_download() --------------->| DownloadManager.start(tasks)
      |                                  |-- for each episode:
      |                                  |     get DRM token
      |                                  |     get Widevine keys
      |                                  |     yt-dlp download video
      |                                  |     yt-dlp download audio
      |                                  |     mp4decrypt video
      |                                  |     mp4decrypt audio
      |                                  |     ffmpeg mux
      |                                  |     queue.put(StatusUpdate)
      |<-- after(100ms, poll_queue) ----|
      |    update row status + bar       |
      |                                  |
      +-- on_cancel() ---------------->| cancel_event.set()
      |                                  |-- checks between phases
      |                                  |-- terminates subprocess

Communication is exclusively through queue.Queue. The UI thread never touches
thread state directly. The background thread never touches tkinter widgets
directly.


7. Self-Update Mechanism
------------------------
1. GET api.github.com/repos/{repo}/releases/latest
2. Compare tag_name (e.g. "v1.2.0") with local __version__
3. If newer:
   a. Show dialog: "v1.2.0 available. Update now?"
   b. Download .exe asset to %TEMP%/SeenShowDL_update.exe
   c. Write seenshow_update.bat:
        @echo off
        timeout /t 2 /nobreak >nul
        del "SeenShowDL.exe.old" 2>nul
        move /y "SeenShowDL.exe" "SeenShowDL.exe.old"
        move /y "%TEMP%\SeenShowDL_update.exe" "SeenShowDL.exe"
        start "" "SeenShowDL.exe"
        del "%~f0"
   d. Launch bat, exit current process
4. On next startup: delete .old file

User data in %APPDATA% is never touched by updates.

9. Build Pipeline
-----------------
On Windows:

    pip install -r requirements.txt
    python build/build.py

build.py does:
  1. Downloads ffmpeg.exe (~80MB) from BtbN/FFmpeg-Builds
  2. Downloads mp4decrypt.exe (~1MB) from Bento4
  3. Runs PyInstaller with seenshow_dl.spec
  4. Output: dist/SeenShowDL.exe (single file, no console)

The spec bundles:
  - All Python code + dependencies (customtkinter, pywidevine, yt-dlp, requests)
  - binaries/ffmpeg.exe and binaries/mp4decrypt.exe
  - customtkinter data files (themes)
  - App icon


10. DRM Pipeline (per episode)
------------------------------
 1. API: GET /{mediaId}/detail           -> episode metadata, fileUrl
 2. API: POST /episodes/{id}/drm-token   -> upfrontToken, MPD URL, userId,
                                            sessionId, merchant
 3. HTTP: GET manifest.mpd               -> extract Widevine PSSH from XML
 4. pywidevine: generate license challenge from PSSH + WVD device
 5. HTTP: POST lic.drmtoday.com/license-proxy-widevine/cenc/
          Headers: x-dt-auth-token, x-dt-custom-data
          Body: challenge bytes
          -> parse license response -> content keys [(kid, key)]
 6. yt-dlp: download bestvideo -> *_video_enc.mp4
 7. yt-dlp: download bestaudio -> *_audio_enc.m4a
 8. mp4decrypt --key kid:key   -> *_video_dec.mp4, *_audio_dec.m4a
 9. ffmpeg -c copy             -> final.mp4
10. cleanup temp files


11. How to Run
--------------
From source (dev/testing):

    cd seenshow-gui
    python -m venv venv

    # Windows
    venv\Scripts\pip install -r requirements.txt
    python run.py

    # Linux / macOS
    ./venv/bin/pip install -r requirements.txt
    python run.py

As built exe (Windows end user):
  1. Double-click SeenShowDL.exe
  2. Sign in with SeenShow credentials
  3. Paste URL -> Fetch -> Download Selected

Note: A Widevine CDM device (.wvd) file is required. Place it in the
app data directory or set the SEENSHOW_WVD_B64 environment variable.
